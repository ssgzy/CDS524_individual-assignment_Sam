### 3. 训练过程

#### 3.1 Level 1 & 2 训练
**参考文件**: `train_fixed.py`, `FINAL_SUMMARY.md`

**训练流程**:
```
1. 初始化环境和Agent
   - 文件: train_fixed.py 第29-41行
   - 配置: lr=1e-4, batch_size=48, curiosity=True

2. 训练循环
   - 文件: train_fixed.py 第60-85行
   - 每4步训练一次（优化）

3. 评估和保存
   - 文件: train_fixed.py 第87-120行
   - 每100轮评估，每200轮保存
```

**关键数据**:
- Level 2训练1000轮
- 训练时间：6分钟
- 最终成功率：100%
- 平均步数：15步

#### 3.2 性能优化
**参考文件**: `FINAL_REPORT.md` (优化部分)

**优化措施**:
1. **MPS GPU加速**
   - 文件: `ai/rainbow_agent.py` 第48-56行
   - 效果: 基础加速

2. **训练频率优化**
   - 文件: `train_fixed.py` 第68-70行
   - 从每步训练改为每4步训练
   - 效果: 8.7倍提速

3. **Batch size优化**
   - 从64降到48
   - 平衡速度和效果

4. **总体效果**
   - 速度: 0.27 → 1.0 eps/s
   - 提升: 3.7倍

### 4. Level 3 挑战

#### 4.1 问题发现
**参考文件**: `LEVEL3_ANALYSIS.md`

**关键发现**:
- 模型陷入重复动作循环（98% RIGHT）
- 复杂度增加100倍
- 局部视野限制

#### 4.2 改进尝试

**尝试1: 迁移学习**
**参考文件**: `train_level3_transfer.py`, `LEVEL3_STRATEGY.md`

**策略**:
- 从Level 2权重开始
- 重置探索率（epsilon=0.5）
- 加倍好奇心（curiosity_weight=1.0）

**尝试2: 增强奖励函数**
**参考文件**: `ai/level3_enhanced_env.py`, `LEVEL3_STRATEGY.md`

**8种奖励信号**:
```python
# 文件: ai/level3_enhanced_env.py
1. Portal使用: +20
2. 箱子放置: +100
3. 距离改进: +2/unit
4. 探索新位置: +1
5. 重复动作: -5
6. 成功: +200
7. 死锁: -50
8. 超时: -20
```

#### 4.3 失败原因分析
**参考文件**: `FINAL_REPORT.md` (深层原因分析)

**理论分析**:
1. **状态空间复杂度**
   - Level 2: ~10^4 状态
   - Level 3: ~10^6 状态
   - 增加100倍

2. **探索难度**
   - 随机探索概率: 1/5^50 ≈ 10^-35
   - 需要更强算法（HER）

3. **信用分配问题**
   - Portal使用后50步才成功
   - 难以建立因果关系
