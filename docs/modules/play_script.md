# 模块说明：`play.py`

## 1. 模块作用
`play.py` 提供可视化交互入口，支持手动游玩和模型自动演示两种模式。

## 2. 核心功能
- Human mode：键盘控制移动/进入子关卡
- Agent mode：加载模型自动决策
- 实时渲染环境状态与UI信息
- 每局结束自动重置并打印结果

## 3. 使用到的知识点
- 人机交互（事件循环）
- 强化学习策略在线执行
- 可视化演示与调试

## 4. 参数说明（可调性 + 意义）

| 参数 | 是否可调 | 意义 |
|---|---|---|
| `--level` | 可调 | 初始关卡 |
| `--mode` | 可调 | `human` 或 `agent` |
| `--model` | 可调 | agent模式下加载的模型路径 |
| `--fps` | 可调 | 渲染刷新率 |
| `--width/--height` | 可调 | 窗口分辨率 |

## 5. 关键运行变量

| 变量 | 是否可调 | 意义 |
|---|---|---|
| `state` | 运行时变量 | 当前状态向量 |
| `episode_reward` | 运行时变量 | 当前局累计奖励 |
| `episode_steps` | 运行时变量 | 当前局已走步数 |
| `info` | 运行时变量 | 终局信息（success/deadlock/timeout） |

## 6. 与作业要求对应
- 满足“UI交互与演示视频”要求
- 可直接用于录制 YouTube Demo
