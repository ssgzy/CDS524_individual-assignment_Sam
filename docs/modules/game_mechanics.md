# 模块说明：`game/mechanics.py`

## 1. 模块作用
`mechanics.py` 封装特殊机制逻辑：传送门、压力板/门状态联动、门阻挡判断。

## 2. 核心功能
- `apply_portal(entity_pos, portals)`：实体踏上传送门时瞬移到配对出口
- `update_pressure_plates(...)`：根据玩家/箱子占位更新压力板与门开关状态
- `is_gate_blocking(pos, gates)`：判断某位置是否被关闭的门阻挡

## 3. 使用到的知识点
- 离散网格规则仿真
- 有限状态机（门开/关）
- 多实体碰撞与占位逻辑

## 4. 变量说明（可调性 + 意义）

| 变量 | 是否可调 | 意义 |
|---|---|---|
| `entity_pos` | 运行时变量 | 待检查实体位置（玩家或箱子） |
| `portals` | 关卡可调 | 传送门对列表，决定空间映射 |
| `player_pos` | 运行时变量 | 玩家位置，用于检测压力板激活 |
| `box_positions` | 运行时变量 | 箱子位置集合，影响压力板持续激活 |
| `pressure_plates` | 关卡可调 | 压力板配置（含 `gate_ids`） |
| `gates` | 关卡可调 + 运行时 | 门实体列表，`is_open` 会动态更新 |
| `pressed_gate_ids`（局部） | 不需调参 | 本帧被激活的门ID集合 |

## 5. 与作业要求对应
- 完整覆盖“传送门机制、压力板与机关门机制”
- 支持玩家与箱子都能触发机关
